## üìù Documentation Standards

### File Naming
- Use `UPPERCASE-WITH-HYPHENS.md` for important/reference docs
- Use `lowercase-with-hyphens.md` for technical/implementation docs
- Include dates in deployment/session docs: `YYYY-MM-DD-description.md`

### Document Structure
All documents should include:
1. **Title** - Clear, descriptive title
2. **Last Updated** - Date of last update
3. **Status** - Current status (Active, Archived, Deprecated)
4. **Purpose** - What this document is for
5. **Audience** - Who should read this
6. **Content** - Well-organized content with headers
7. **Related Docs** - Links to related documentation

### Archiving Policy
Documents should be archived when:
- Feature is fully implemented and stable
- Deployment is complete and superseded
- Session/sprint is complete
- Information is outdated but worth preserving
